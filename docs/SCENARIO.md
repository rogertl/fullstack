# 场景描述文档

> **项目**: 合同管理系统（华道）
> **阶段**: Stage -1 场景阶段
> **日期**: 2026-06-08

---

## 1. 目标用户

华道公司内部员工，包括：

- **系统管理员**：负责系统配置、用户管理、权限分配
- **业务人员**：负责合同的创建、编辑、提交审批
- **部门领导**：负责合同审批
- **普通员工**：查看自己的合同信息

---

## 2. 核心功能（前3个）

### 功能 1：用户认证与权限管理

- 登录/登出（JWT 双 token 模式）
- 用户管理（创建、编辑、删除用户）
- 部门管理（组织架构管理）
- 角色权限管理（管理员、普通用户等）

### 功能 2：菜单权限控制

- 基于角色的菜单显示
- 系统管理菜单（用户、部门、角色管理）
- 合同管理菜单（创建合同、查看本人合同、查看全部合同）

### 功能 3：数据权限隔离

- 普通用户：只能查看自己的数据
- 管理员：可以查看所有数据
- 数据权限通过后端过滤，前端无法绕过

---

## 3. 非功能性需求

### 安全性

- JWT 双 token 认证（Access Token 15分钟，Refresh Token 7天）
- 所有 API 请求必须携带有效 token
- Token 过期自动刷新，刷新失败跳转登录页

### 权限控制

- **功能权限**：菜单级别控制（有权限才能看到菜单项）
- **数据权限**：数据级别控制（只能看自己的数据，管理员看全部）
- 后端严格验证，前端隐藏不等于权限控制

### 用户体验

- 使用 Ant Design 组件库，提供一致的用户体验
- 响应式布局，支持不同屏幕尺寸
- 操作反馈明确（成功/失败提示）

### 可维护性

- 代码结构清晰，模块化设计
- 遵循契约优先开发原则（Zod Schema 唯一真相来源）
- 完整的类型安全（TypeScript + Zod）

## 6. 系统菜单结构

```
合同管理系统
├── 系统管理
│   ├── 用户管理
│   ├── 部门管理
│   └── 角色权限
└── 合同管理
    ├── 创建合同
    ├── 我的合同
    └── 全部合同（仅管理员可见）
```

---

## 7. 角色与权限定义

### 角色 1：系统管理员

- **功能权限**：所有菜单和功能
- **数据权限**：查看所有数据

### 角色 2：普通用户

- **功能权限**：
  - ✅ 合同管理 → 创建合同
  - ✅ 合同管理 → 我的合同
  - ❌ 合同管理 → 全部合同
  - ❌ 系统管理（所有菜单）
- **数据权限**：只能查看自己的数据
