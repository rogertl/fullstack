# 契约开发架构 - 项目启动问卷系统

## 设计原则

1. **渐进式问题** - 基础问题 → 技术选型 → 开发约束
2. **智能分支** - 某些答案会影响后续问题
3. **默认值优先** - 80%项目用标准配置，20%需要定制
4. **可逆性** - 问卷后可预览配置，允许返回修改

---

## 问卷结构

### 第一部分：项目基础信息（必答）

#### Q1. 项目名称
**类型**: 文本输入
**默认**: 未命名项目
**用途**:
- package.json `name`
- README.md `title`
- 数据库名称

#### Q2. 项目描述
**类型**: 多行文本
**可选**: 是
**用途**: README.md 描述

#### Q3. 项目类型
**类型**: 单选
**选项**:
- A. 完整全栈项目（前端+后端）
- B. 纯后端API
- C. 纯前端项目（调用外部API）
- D. 验证/测试项目（学习架构）

**影响**:
- A: 完整7阶段流程
- B: 跳过前端阶段（Stage 4）
- C: 只执行前端阶段
- D: 简化配置，快速启动

---

### 第二部分：技术栈选择

#### Q4. 后端框架（仅Q3选择A或B时出现）
**类型**: 单选
**选项**:
- A. Express（推荐，生态成熟）
- B. Fastify（性能优先）
- C. Koa（轻量简洁）
- D. NestJS（企业级，TypeScript原生）
- E. 自定义

**默认**: A

#### Q5. 前端框架（仅Q3选择A时出现）
**类型**: 单选
**选项**:
- A. React + Vite（推荐，快速开发）
- B. Vue 3 + Vite
- C. Next.js（SSR需求）
- D. Svelte（性能优先）

**默认**: A

#### Q6. 数据库
**类型**: 单选
**选项**:
- A. SQLite（开发环境，快速启动）
- B. PostgreSQL（生产推荐）
- C. MySQL
- D. MongoDB（文档型）
- E. 无数据库（纯计算/代理）

**默认**: A（开发）/ B（生产）

#### Q7. ORM选择（仅Q6选择A/B/C时出现）
**类型**: 单选
**选项**:
- A. Prisma（推荐，类型安全）
- B. Drizzle（轻量，类似SQL）
- C. TypeORM（类似Java Hibernate）
- D. MikroORM（DCO友好）
- E. 纯SQL（无ORM）

**默认**: A

#### Q8. 是否需要实时功能？
**类型**: 多选
**选项**:
- [ ] WebSocket（实时通信）
- [ ] Server-Sent Events（单向推送）
- [ ] 轮询（简单场景）
- [ ] 不需要

**影响**: 添加 WebSocket 依赖配置

---

### 第三部分：架构约束

#### Q9. 是否需要CQRS/ES？
**类型**: 单选
**选项**:
- A. 完整CQRS + Event Sourcing（复杂领域）
- B. 仅CQRS（读写分离）
- C. 传统架构（不分离）
- D. 不确定，帮我判断

**默认**: D（由AI判断）

**判断依据**:
- 复杂业务规则 → A
- 读多写少 → B
- 简单CRUD → C

#### Q10. 是否需要领域驱动设计（DDD）？
**类型**: 单选
**选项**:
- A. 是（完整DDD：聚合根、值对象、领域服务）
- B. 轻量DDD（只有聚合和值对象）
- C. 否（传统分层架构）

**默认**: B

#### Q11. 认证方式？
**类型**: 单选
**选项**:
- A. JWT（推荐，无状态）
- B. Session（传统）
- C. OAuth 2.0（第三方登录）
- D. 不需要认证

**默认**: A

## 使用流程

1. **启动问卷**: `npm run init-project` 或 `./scripts/questionnaire.sh`
2. **回答问题**: 系统根据前面答案动态调整后续问题
3. **预览配置**: 展示将生成的配置摘要
4. **确认生成**: 运行初始化脚本
5. **项目就绪**: 开始开发
